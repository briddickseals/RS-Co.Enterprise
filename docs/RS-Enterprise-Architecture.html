Enterprise Operating Model
Governance structure, taxonomy standards, and operational principles for running the RS&Co. platform ecosystem at institutional scale.

Governance Principle

Single Source of Truth Architecture

Every data entity has exactly one system of record. Wix CMS owns deal data. SharePoint owns documents. Business Central owns financials. Dynamics 365 owns contacts and pipeline. All other systems consume from these sources via Power Automate or API, never duplicate. If a data point can be derived, it is calculated on read, not stored redundantly.

File & Document Taxonomy

Uniform Naming Convention

All files follow: [Entity]-[Category]-[Description]-[Version].[ext]. SharePoint enforces 9 top-level categories (Executive, Corporate, Development, Projects, Brands, Marketing, Ventures, People, Archive). Deal documents use /Deals/{dealSlug}/{category}/ structure. No loose files at root. No personal naming conventions.

Agent Deconfliction Protocol

Zero Redundancy

Each agent has a primary domain and platform. Claude.ai handles strategy and architecture. Claude Code handles engineering and deployment. OpenAI handles content and analysis. Copilot Studio handles client-facing interactions. Power Automate handles event-driven workflows. No two agents perform the same function on the same platform.

Vertical Integration Model

RS&Co. Brand Hierarchy

All verticals (Enterprise, Acumen, Advisory, Apeiron, Apogee, AREAops, LUCY) share the same infrastructure stack. Each vertical has its own Wix CMS collections, SharePoint subfolder, and CRM pipeline. Shared services include authentication, payment processing, email automations, and analytics. Build once, deploy across all brands.

Security & Access Model

Role-Based Access Control

Four-tier access model: Admin (full control), Escrow (deal-party access), NDA (investor-level), Public (anonymous). Enforced consistently across Wix Members, SharePoint groups, Dynamics 365 security roles, and Copilot Studio agent access checks. All authentication flows through Entra ID for Microsoft services, Wix Members for client-facing.

Continuous Improvement Cycle

Weekly Cadence

Monday: Power BI dashboard review, pipeline health check. Wednesday: Agent performance review, workflow error audit. Friday: Backlog grooming in Azure DevOps, sprint planning for next week. Monthly: Architecture review, decommission progress, vendor cost analysis. Quarterly: Strategic alignment review, roadmap update.
