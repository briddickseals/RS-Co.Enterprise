Enterprise Operating Model
Governance structure, taxonomy standards, and operational principles for running the RS&Co. platform ecosystem at institutional scale. These principles ensure consistency, security, and efficiency across all systems.

üìê Governance Principle

Single Source of Truth Architecture

Every data entity has exactly one system of record. Wix CMS owns deal data. SharePoint owns documents. Business Central owns financials. Dynamics 365 owns contacts and pipeline. All other systems consume from these sources via Power Automate or API, never duplicate. If a data point can be derived, it is calculated on read, not stored redundantly.

Golden Rule: If you're unsure where data lives, check the system of record. If you're creating new data, ensure it's in the correct system of record from the start.

Uniform Naming Convention

All files follow: [Entity]-[Category]-[Description]-[Version].[ext]. SharePoint enforces 6 top-level categories (Executive, Corporate, Brands, Marketing, People, Projects, Archive). Deal documents use /Deals/{dealSlug}/{category}/ structure. No loose files at root. No personal naming conventions.

ü§ñ Agent Deconfliction Protocol

Zero Redundancy

Each agent has a primary domain and platform. Claude.ai handles strategy and architecture. Claude Code handles engineering and deployment. OpenAI handles content and analysis. Copilot Studio handles client-facing interactions. Power Automate handles event-driven workflows. No two agents perform the same function on the same platform.

Rule: Before assigning a task to an agent, verify no other agent owns that domain. When in doubt, escalate to Chief Strategy Agent (Claude) for routing.

üè¢ Vertical Integration Model

RS&Co. Brand Hierarchy

All verticals (Enterprise, Acumen, Advisory, Apeiron, Apogee, AREAops, LUCY) share the same infrastructure stack. Each vertical has its own Wix CMS collections, SharePoint subfolder, and CRM pipeline. Shared services include authentication, payment processing, email automations, and analytics. Build once, deploy across all brands.

Role-Based Access Control
